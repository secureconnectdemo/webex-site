üöÄ AI Webex Assistant Deployment & Automation Guide

Build, deploy, and scale your Webex bot using GitHub, Render, and Google APIs.

üîß Component Overview

Component Purpose
GitHub üóÇÔ∏è Stores your source code and version history
- Think of GitHub as your code‚Äôs home base.
- All your bot logic (like bot.js, sheet.js, package.json, etc.) lives here.
- Every change is tracked, and it integrates directly with Render for auto-deploy.
Render üöÄ Runs your bot as a live web service
- Render acts like your bot‚Äôs engine room.
- It pulls the code from GitHub and hosts it in a Node.js environment.
- Provides a public HTTPS endpoint (e.g., https://yourbot.onrender.com/webhook) where Webex can send events.
- Executes bot.js to process each event.
Webex üîî Triggers your bot with real-time events
- Webex sends webhook events to your Render-hosted /webhook endpoint whenever something happens (like a new message in a room).
- Your bot must be registered in the Webex Developer Portal with this endpoint.

üì• Bot Flow

Component Action
WebexSends a messages event to Render
RenderReceives it at /webhook, runs bot.js
bot.jsParses /handoff, calls sheet.js.appendRow()
sheet.jsAuthenticates with Google API and writes to Sheet
Google SheetUpdates with new customer handoff info
Webex (optional)Sends confirmation reply back in the chat

üíª Webex Developer Setup

  1. Go to the Webex Developer Portal ‚Äì My Apps
  2. Click Create a New App ‚Üí Bot
  3. Fill in the following fields:
    • Name: AI Webex Assistant
    • Bot Username: Must end with @webex.bot
    • Icon: Upload an image
    • Description: e.g., "Automates CS handoffs, flags risk, and more"
  4. Click Add Bot and copy the Bot Access Token
  5. Use this token in your Render environment as WEBEX_ACCESS_TOKEN

üîê GitHub Setup

  1. Go to GitHub and sign in or sign up.
  2. Click New Repository from your dashboard.
  3. Name it (e.g., ai-webex-assistant).
  4. Set visibility to Public or Private, depending on your team needs.
  5. Click Create Repository.

üìä Google Sheets Integration

  1. Go to the Google Cloud Console
  2. Create a new project (or select an existing one)
  3. Enable the following APIs:
    • ‚úÖ Google Sheets API
    • ‚úÖ Google Drive API (required for file-level permissions)
  4. Go to IAM & Admin > Service Accounts
  5. Click Create Service Account
    • Name it something like sheets-access-bot
  6. After creating it:
    • Go to the Keys tab
    • Click Add Key ‚Üí Create new key ‚Üí Choose JSON
    • This will download a .json file ‚Äî save it safely
    • Use its contents as your GOOGLE_SERVICE_ACCOUNT variable in Render

üìÑ Share Access to the Sheet

  1. Open the Google Sheet you want to update via the bot
  2. Click Share, and give Editor access to the service account‚Äôs email
    Example: sheets-access-bot@yourproject.iam.gserviceaccount.com

üß† Sample sheet.js Code (Minimal, Clean, Working)


  const { google } = require('googleapis');
  
  const auth = new google.auth.GoogleAuth({
    credentials: JSON.parse(process.env.GOOGLE_SERVICE_ACCOUNT),
    scopes: ['https://www.googleapis.com/auth/spreadsheets'],
  });
  
  const sheets = google.sheets({ version: 'v4', auth });
  
  async function appendRow(spreadsheetId, values) {
    try {
      await sheets.spreadsheets.values.append({
        spreadsheetId,
        range: 'Sheet1!A1',
        valueInputOption: 'USER_ENTERED',
        requestBody: {
          values: [values], // e.g., ['CustomerABC', 'ARR $100K']
        },
      });
      console.log("‚úÖ Row appended successfully");
    } catch (error) {
      console.error("‚ùå Google Sheets append error:", error.message);
    }
  }
  
  module.exports = { appendRow };

‚öôÔ∏è Render Deployment

  1. Go to Render and sign in or create a new account.
  2. Click New ‚Üí Web Service from your Render dashboard.
  3. Under Connect a Repository, click Authorize GitHub (if you haven‚Äôt already).
  4. Grant access to the repo where your Webex Assistant bot code lives.
  5. Select the repository from the list and proceed to configuration.

‚öôÔ∏è Configure the Web Service in Render

On the configuration screen in Render:

  • Name: Give your service a unique name (e.g., ai-webex-bot).
  • Region: Choose the region closest to your users (e.g., Oregon (US West) or Frankfurt (EU Central)).
  • Branch: Select main (or whichever branch contains your working code).
  • Build Command: 
    npm install
  • Start Command: 
    node bot.js
  • Environment: Select Node from the dropdown.

üîê Set Environment Variables in Render

In the Environment section of your Render Web Service setup, add the following variables:

Key Value Description
WEBEX_ACCESS_TOKEN Your bot's access token from the Webex Developer Portal
GOOGLE_SERVICE_ACCOUNT The full contents of your downloaded service-account.json file
‚úÖ Important Notes:
- Never hardcode credentials in your codebase.
- Ensure credentials.json is ignored via .gitignore.

üöÄ Deploy the Service

  1. After entering your build/start commands and environment variables, click Create Web Service.
  2. Render will now:
    • üîÅ Clone your GitHub repo
    • üì¶ Install dependencies using your package.json
    • üöÄ Launch your app using the specified start command
  3. Once deployment is complete, your bot will be live at the public HTTPS endpoint Render provides (e.g., https://yourbot.onrender.com/webhook).

üåê Webhook Registration

‚úÖ Option 1: GUI (Webex Developer Portal)

Follow these steps to register your bot webhook using the Webex Developer Portal:

  1. Visit the Webex Developer Portal
  2. Log in with your Webex credentials.
  3. Click Create a Webhook
  4. Fill out the form with the following values:
  • Name: e.g., Webex Bot - ECS
  • Target URL: https://yourbot.onrender.com/webhook
  • Resource: messages
  • Event: created
  • Secret: (optional but recommended for validating payloads)

5. Click Run to create the webhook.

üí° Tip: For production environments, we recommend using the API or Node.js script to ensure consistent, repeatable setup ‚Äî especially during ECS or Render redeployments.

üîí Permissions & Testing

Ensure your Webex bot has the following OAuth scopes enabled in the Developer Portal:

  • spark:messages_read ‚Äî required to read incoming messages
  • spark:messages_write ‚Äî required to send messages
  • spark:rooms_read ‚Äî optional, allows room details access
  • spark:people_read ‚Äî optional, allows identifying users by email or ID
‚ö†Ô∏è Missing any of these may prevent the bot from reading or responding to messages properly.
